# help 指令 · 设计草稿

- **目标**：提供一条用户可在群聊/私聊中发送的指令，用于查看 Bot 使用说明与各插件用户文档，无需离开聊天环境即可获取帮助。
- **草稿日期**：2026-02-14

---

## 背景与目标

- 希望用户发送如「help」或「帮助」即可看到总览；发送「help 插件名」即可看到该插件的使用说明。
- **帮助内容仅来自运行时代码**：由各插件在实例上提供 `meta.help.markdown`（见项目规约「插件 meta 与 help 文本」）。**不得**在运行中读取 `docs/` 目录——docs 仅作项目文档，不是执行期的一部分。

---

## 指令约定

### 触发词

- **主触发词**：`help`、`帮助`（任选其一或均支持）。
- **可选**：斜杠形式 `/help`、`/帮助`，与框架命令风格统一时可一并支持。
- **严格匹配**：遵循项目规约「指令触发：严格匹配」，strip 后整句为 `help` / `帮助` 或其后仅跟空格/参数，避免误触发。

### 行为

| 用户发送 | 行为 |
|----------|------|
| `help` / `帮助` | 回复**总览**：Bot 简介 + 已支持的插件列表 + 提示「发 `help 插件名` 查看该插件说明」。 |
| `help 插件名` / `帮助 插件名` | 回复**该插件的用户说明**正文；若插件名无法识别则提示「未找到该插件说明」并列出可用插件名。 |

### 插件名与列表来源

- **总览**中的「插件名」与「发 help 插件名」中的**插件名**一致，便于用户复制。
- 插件列表来源：已加载的插件实例；展示名/别名由 help 插件配置或从插件 `meta` / `name` 取得。
- 仅当插件提供 `meta.help.markdown` 时，才在「发 help 插件名」时返回详细说明正文；否则仅列出插件名或提示无详细说明。

---

## 帮助内容来源与格式

- **总览**：可写死一段简短文案（如「本 Bot 支持以下插件：…」）+ 从已加载插件生成的列表（展示名可来自插件 `name` 或 help 插件配置的别名）。
- **插件说明**：**仅**从插件实例的 **`meta.help.markdown`** 读取；不读取任何 `docs/` 下文件。
- 回复时可将 Markdown 转为**纯文本**（去标题记号、表格简化为行、去掉链接等），以适配 QQ 消息展示；若平台支持富文本，可保留部分格式。
- **长度限制**：单条消息有长度上限时，可截断并注明「（内容过长，已截断）」或拆成多条连续发送；优先保证「总览」简短、「插件说明」按段或按字数分包。

---

## 权限与范围

- **权限**：所有用户（群聊与私聊）均可使用，无需管理员。
- **范围**：群聊、私聊均响应（或按需仅群聊/仅私聊，实现时在 handler 上做过滤）。

---

## 实现要点（供后续编码参考）

1. **归属**：以**独立插件**（如 `help_plugin` / `help`）实现，或作为项目内置能力挂到现有入口；推荐独立插件便于关闭/扩展。
2. **取插件帮助文本**（符合项目规约「插件 meta 与 help 文本」）：**仅**从插件实例的 **`meta.help.markdown`** 属性读取帮助正文（Markdown 字符串）。**不得**读取 `docs/` 目录——docs 不是执行期的一部分。若 `meta.help.markdown` 不存在，则仅展示插件 name/description 或提示无详细说明。
3. **配置**：插件可配置「总览文案」、展示名/别名（插件名 → 显示名），不涉及任何 `docs/` 路径。
4. **触发**：`help` / `帮助` 整句或带一个参数（插件名），解析时 strip，参数用空格分隔；符合规约「严格匹配」：无参数即总览，有参数即查插件。
5. **总览内容**：列出已加载的、可展示的插件（展示名），并提示「发送 `help 插件名` 查看详细说明」。

---

## 与规约的关系

- **插件 meta**：遵循项目规约「插件 meta 与 help 文本」——插件实例提供 `meta` 字段，help 插件**仅**从 `Plugin.meta.help.markdown` 取帮助正文；不读取 `docs/`。
- 各插件若需在 help 中展示说明，应在代码中设置 `meta.help.markdown`（如从包内资源或常量加载）；项目中的 `docs/plugins/*_用户说明.md` 仅作给人看的文档，可经构建或人工同步到插件的 `meta.help.markdown`，但运行时代码不读 docs。
